# data/ — Directory Structure and Data Provenance

This directory holds all input and intermediate data for the CCZ connectivity pipeline.
Large files (NetCDF, Zarr, shapefiles) are not tracked in git (see `.gitignore`).

---


## Data Sources and Provenance

### HYCOM (Hybrid Coordinate Ocean Model)
- **Source:** HYCOM Global Analysis System, Experiment 93.0
- **OPeNDAP URL:** `https://tds.hycom.org/thredds/dodsC/GLBy0.08/expt_93.0`
- **Variables:** `water_u_bottom`, `water_v_bottom`
- **Spatial subset:** 190°E–250°E (= 170°W–110°W), 5°S–25°N (HYCOM convention)
- **Temporal coverage:** Jan 2019, Jul 2019, Jan 2023, Jul 2023 (69-day windows)
- **Downloaded by:** `workflow/00_download_hycom.py`

### CCZ Boundary
- **Source:** ISA (International Seabed Authority) GIS data
- **Description:** Outer polygon of the Clarion-Clipperton Zone study domain
- **Place in:** `data/shapefiles/ccz_boundary.shp`

### APEI Polygons
- **Source:** ISA — Areas of Particular Environmental Interest
- **Description:** 9 APEIs; each polygon has a `Remarks` field with label
  (e.g. "APEI-1", "APEI-7", etc.)
- **Place in:** `data/shapefiles/apei.shp`

### AMI Polygons
- **Source:** ISA contractor exploration block polygons
- **Description:** Deep-sea mining licence areas used in the mined scenario
- **Place in:** `data/shapefiles/ami.shp`

### CCZ Grid
- **Generated by:** `src/ccz_connectivity/grid.py` → `make_ccz_grid()`
- **Description:** 100 km × 100 km equal-area grid in EPSG:8857, clipped to
  CCZ boundary, reprojected to EPSG:4326 for output.  FID column is the
  unique polygon identifier used throughout the pipeline.
- **Centroids:** `data/shapefiles/ccz_grid_centroids.csv` — columns:
  `polyid` (= FID), `lon`, `lat`

## Generating the Grid

After placing `ccz_boundary.shp` in `data/shapefiles/`, generate the grid
with:

```python
import yaml
from ccz_connectivity.grid import make_ccz_grid, save_grid
from pathlib import Path

cfg = yaml.safe_load(open("config/config.yaml"))
gdf = make_ccz_grid(cfg)
save_grid(
    gdf,
    out_shp=Path("data/shapefiles/ccz_grid.shp"),
    out_centroids=Path("data/shapefiles/ccz_grid_centroids.csv"),
)
```

---

## Notes

- All shapefiles should be in WGS84 (EPSG:4326) for compatibility with the pipeline.
- HYCOM data are downloaded in HYCOM's 0–360° longitude convention and
  converted to −180–180° by `workflow/00_download_hycom.py`.
- Zarr stores use the OceanParcels 3.0+ schema: see the Zarr schema
  section of the manuscript supplement or the `tests/generate_test_data.py`
  script for a complete specification.
